# برومبت إصلاح مصدر الأسئلة

## المهمة

أعد كتابة `prisma/seed.ts` ليحلّل الأسئلة مباشرةً من الملف النصي `مسابقة الحروف.txt`
بدلاً من الملف `موسوعة_مسابقة_الحروف_منقحة.json` الذي يحتوي على
إجابات خاطئة مصطنعة بالذكاء الاصطناعي.

---

## السياق

مسار الملف: `مسابقة الحروف.txt` (في جذر المشروع)  
هذا الملف هو المصدر الرسمي — كل سؤال فيه يحمل الإجابات الخاطئة الأصلية المقصودة.

---

## الملاحظات الهيكلية للملف

### ١. رؤوس الأقسام (كل حرف يبدأ هكذا)

```
## مسابقة حرف ( أ )
## مسابقة حرف ( ث )
## مسابقة حرف (و)     ← بدون مسافات أحياناً
```

استخرج اسم الحرف بالتعبير النمطي: `/## مسابقة حرف \(\s*(.+?)\s*\)/`

---

### ٢. أقسام مكررة — يجب أخذ النسخة الأولى فقط

الحروف التالية تظهر مرتين في الملف — **تجاهل النسخة الثانية**:

| الحرف | السطر الأول | السطر المكرر |
|:---:|:---:|:---:|
| خ | 787 | 849 |
| ف | 1631 | 2144 |
| ق | 1665 | 2178 |
| ك | 1699 | 2212 |
| ل | 1733 | 2246 |
| م | 1767 | 2280 |

الحل: تتبّع الحروف التي رُصدت بالفعل، وعند تكرار الحرف بعد قراءة 10+ أسئلة منه → أوقف الإضافة.

---

### ٣. تنسيقات الأسئلة — خمسة أنماط

#### ● النمط أ — أحرف: أ ب ت (سطر 3 – 554)

```
**سN: نص السؤال؟**
* خيار أول
* خيار صحيح (الإجابة الصحيحة)
* خيار ثالث
* خيار رابع
```

- **السؤال**: يطابق `/^\*\*س\d+:\s*(.+?)\*\*\s*$/`
- **الخيار**: يبدأ بـ `* ` (نجمة مسافة)
- السطر الفارغ يفصل بين الأسئلة

---

#### ● النمط ب — أحرف: ث ج ح خ د ذ ر ز س ش ص ض (سطر 555 – 1630)

```
* نص السؤال؟
   أ) خيار أول
   ب) خيار صحيح (الإجابة الصحيحة)
   ج) خيار ثالث
   د) خيار رابع
```

- **السؤال**: يبدأ بـ `* ` وينتهي بـ `?` أو `؟`
- **الخيار**: يطابق `/^\s+[أبجد]\)\s*(.+)/`

---

#### ● النمط ج — أحرف: ف ق ك ل م (سطر 1631 – 1801)

```
نص السؤال؟
1. خيار أول
2. خيار صحيح (الإجابة الصحيحة)
3. خيار ثالث
4. خيار رابع
```

- **السؤال**: سطر ينتهي بـ `?` أو `؟` وليس مسبوقاً برقم
- **الخيار**: يطابق `/^\d+\.\s*(.+)/`

---

#### ● النمط د — أحرف: ع غ (سطر 1802 – 2143)

```
N. نص السؤال؟
- خيار أول
- خيار صحيح (الإجابة الصحيحة)
- خيار ثالث
- خيار رابع
```

- **السؤال**: يطابق `/^\d+\.\s*(.+[؟?])\s*$/`
- **الخيار**: يبدأ بـ `- `

---

#### ● النمط هـ — أحرف: ن هـ (سطر 2314 – 2699)

```
سN
نص السؤال؟
أ) خيار أول
ب) خيار صحيح (الإجابة الصحيحة)
ج) خيار ثالث
د) خيار رابع
```

- سطر `سN` (رقم عربي أو لاتيني) يُعلن بداية سؤال جديد — يطابق `/^س\d+$/` أو `/^س[٠-٩]+$/`
- **السؤال**: السطر التالي مباشرة
- **الخيار**: يطابق `/^[أبجد]\)\s*(.+)/`

---

#### ● النمط و — أحرف: و ي (سطر 2700 – النهاية)

```
***

**سN: نص السؤال؟**
أ) خيار أول
ب) خيار صحيح (الإجابة الصحيحة)
ج) خيار ثالث
د) خيار رابع
```

- الفاصل `***` يعلن بداية سؤال جديد
- **السؤال**: يطابق `/^\*\*س[٠-٩٠-9]+:\s*(.+?)\*\*\s*$/` (أرقام عربية أو لاتينية)
- **الخيار**: يطابق `/^[أبجد]\)\s*(.+)/`

---

### ٤. استخراج الإجابة الصحيحة

في جميع الأنماط، الإجابة الصحيحة علامتها: `(الإجابة الصحيحة)` في نهاية نص الخيار.  
بعد تحديد الـ `correctIndex`، احذف هذه العلامة من نص الخيار.

```ts
const ci = options.findIndex(o => o.includes('(الإجابة الصحيحة)'));
if (ci === -1) { /* تجاهل السؤال */ }
const cleanOptions = options.map(o => o.replace(/\s*\(الإجابة الصحيحة\)\s*/, '').trim());
```

---

### ٥. حروف غير موجودة في الملف النصي

الحرفان **ط** و**ظ** غير موجودَين في `مسابقة الحروف.txt` — هذا طبيعي، تجاهلهما.

---

## خطوات التنفيذ

1. **اقرأ** `prisma/seed.ts` الحالي بالكامل
2. **اقرأ** أول 100 سطر من `مسابقة الحروف.txt` لتأكيد التنسيق
3. **أعد كتابة** `prisma/seed.ts`:
   - احذف كل الكود القديم
   - اكتب `parseQuestionsFile(filePath)` يتعامل مع الأنماط الخمسة
   - استخدم مسار: `path.join(__dirname, '..', 'مسابقة الحروف.txt')`
4. **شغّل**: `npx prisma db seed`
5. **تحقق** من المخرجات: يجب أن يطبع عدد أسئلة كل حرف — لا يجب أن يكون أي حرف بـ 0
